What You're Seeing

This dashboard shows a live end-to-end ML pipeline built on MongoDB Atlas as the Operational Data Store. 50 simulated cell towers generate network telemetry at 17,000+ events/second into Kafka. Apache Flink (PyFlink) applies a 5-minute sliding window with 30-second slides per cell tower, computing rolling statistics (avg, min, max, p95) and writing to Atlas every 30 seconds. An Atlas Database Trigger fires on every insert, calls an MLflow model to classify network health (excellent / good / poor), and writes the prediction back to Atlas — all within milliseconds, zero external orchestration.

ODS Requirements Demonstrated: Real-time ingestion (FR-3), schema flexibility with nested stats (FR-5), API-based access (FR-9), ML feature enablement (FR-11), event-driven operational workflows via Triggers (FR-12), operational scope with raw data in Kafka and aggregates in ODS (FR-14).
